Browsers, Flash and non-latin text input

Although the fairly well know WMODE set to opaque/transparent non-US keyboard bug that existed for many years has been fixed by most browser vendors there remains some territory that has not yet been successfully crossed.

Some non-latin languages (greek, russian, hebrew) still get a rough time with Flash keyboard input when WMODE is set to anything other than window (the default). Hitting a single key yields two garbled characters in Internet Explorer and in Chrome or Firefox the application shows output as if a US keyboard was used.

Below is a simple text box to try things out for yourself. First a flash object with wmode set to default (which is window), second with wmode set to opaque and finally transparent. There should be no difference between opaque and transparent.

wmode = 'window'
